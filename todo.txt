COMPLETED
- logging now warns when module worker threads are still alive during stop/reset cleanup
- startup and initialization logging severity is standardized around debug/info boundaries in the runtime control path
- path columns for generated media are no longer length-limited (`dicos.frame.path`, `dicos.poi_search.crop_dir`)
- input media camera/video frame handoff now uses locks instead of unsynchronized shared state
- failed source starts are handled earlier
-- enabled camera URLs, uploaded videos, and webcams are probed before `/start` publishes a run
-- source validation failures are persisted in `last_error`
-- `/status` and `/system/resources` surface those failures through admission state for the frontend
- ingestor no longer loops forever when no source yields frames
-- run metadata is only persisted after the first real frame arrives
-- ingest publishes an error and stops if frames stop arriving past the configured timeout
- annotated-video second-run startup now clears stale annotation Rabbit queues before the next run
- critical worker-thread exits now propagate to module `error` state instead of silently hanging
- database/rabbitmq/ffmpeg dependency failures now surface in `/status`, `/system/resources`, and the frontend
- ingest bootstrap logs are separated from per-run logs until the first real frame confirms the run
- per-module queue depth and backpressure state now surface in `/status`, `/system/resources`, and monitoring
- mixed-source settings, monitoring, machine-consumable feeds, and built-in API docs are now in the UI
- POI update age is now exposed as elapsed seconds in the API and rendered as wall-clock time in the UI
- automated Docker build validation is available through `scripts/docker_build_test.py`
- POI search page now renders registered searches in a dedicated responsive results grid
- POI result detail page now has a dedicated responsive layout instead of the legacy history-card view
